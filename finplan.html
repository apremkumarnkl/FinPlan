Google Drive backup
⚠ Google sign-in only works when this app is opened over https:// from a domain you've registered in your own free Google Cloud project (not from a local file:// path). See the setup steps below — this is a one-time, five-minute setup that only you control; FinPlan never sees your Google account credentials.
1. One-time setup

In Google Cloud Console: create a project → enable the Google Drive API → create an OAuth Client ID (type: Web application) → add the URL you're hosting this file at (e.g. your GitHub Pages URL) under "Authorized JavaScript origins". Paste the Client ID below.

Saved automatically in this browser (via the "Auto-save to this browser" setting below) so you don't need to re-paste it each time you reopen the app. It's also included in your JSON/Drive backups.
2. Connect & backup
Not connected
Uses the restricted drive.file scope — FinPlan can only see backup files it creates itself, never the rest of your Drive. Once connected, FinPlan keeps itself signed in automatically — silently renewing the connection every few minutes before it can expire, and reconnecting each time you reopen the app — as long as this tab stays open and you're still signed into Google in this browser. Click "Sign out" if you want it to stop.
3. Optional: truly persistent connection (advanced)
⚠ The "Connect & backup" above already keeps itself signed in while this tab is open, and reconnects on reopening if your browser still has an active Google session. But browser-only sign-in fundamentally cannot survive things like a fully closed app, a cleared browser session, or Google's own session policies — there's no way around that without a small backend. This section adds one: a free Cloudflare Worker you deploy yourself that holds your OAuth Client Secret (never your data) and refreshes your connection server-side, so it survives all of that. It's optional — everything above works fine without it.
Set up once, stays connected indefinitely*

Deploy the Worker script from your download package (gdrive-token-proxy-worker.js) to Cloudflare Workers (free tier) — the file has full step-by-step setup comments at the top. Then paste its URL below and click Enable.

⚠ Getting "Error 400: redirect_uri_mismatch"? Google requires the redirect URI below to match, character-for-character, an entry in your OAuth Client's "Authorized redirect URIs" in Google Cloud Console — including trailing slashes and capitalization. Copy the exact value below rather than typing it by hand; a mismatched trailing slash is the single most common cause of this error.
Not enabled — using the session-only connection above.
*How long the connection actually lasts is set by Google, not FinPlan: while your Google Cloud OAuth consent screen is in "Testing" status, Google expires these tokens after about 7 days, requiring one re-click of "Enable" — switch the consent screen to "In production" in Google Cloud Console (no formal app review needed for personal use with this scope) for it to stop expiring altogether.
Live stock/share prices (optional)
Two ways to get this, pick whichever suits you — both are optional and independent.
Option A — No backend needed (Alpha Vantage)

Alpha Vantage's public API can be called directly from the browser — no server needed. Get a free key at alphavantage.co (instant, no credit card).

⚠ The free tier is limited to 25 requests per day, total — each stock fetch (and "Fetch all") counts against that. Fine for occasionally refreshing a handful of holdings; not for large portfolios or frequent updates. Coverage is strongest for US-listed stocks; Indian NSE/BSE symbols may need the .BSE suffix and aren't always available — check Alpha Vantage's own symbol search if a fetch fails.
Option B — Higher volume, needs a one-time backend deploy
⚠ Indian exchanges (NSE/BSE) block direct browser requests without session cookies, and Yahoo Finance's fuller data needs headers browsers can't set — this option relays through a free Cloudflare Worker you deploy yourself, with no daily request cap. It never sees your FinPlan data, only ticker symbols.

Deploy the Worker script from your download package (stock-price-proxy-worker.js) to Cloudflare Workers (free tier) — full setup steps are in the comments at the top of the file. Then paste its URL below.

Once either is set, add a ticker symbol to each stock in its Edit form (e.g. RELIANCE.NS for NSE, SYMBOL.BO/.BSE for BSE, or just AAPL for US stocks) to enable "Fetch Latest Price" on the NAV Update page. If both are configured, Alpha Vantage is tried first since it needs no deployment, falling back to your Worker if that fails. Either way, treat fetched prices as indicative, same as any free quote source.
Auto backup
⚠ Auto-backup only runs while this tab stays open — it can't run in the background after you close it.
Auto-save to this browser

Saves automatically to this browser's local storage, so refreshing or reopening the tab doesn't lose your data. Scoped to this exact browser and page address only — it won't follow you to another device, browser, or file location, and it can't run inside Claude.ai's own preview panel.

Auto-backup to Google Drive

Requires Google Drive to be connected above. Keeps a single file, finplan_autobackup.json, up to date in your Drive. Works on mobile/Android too, since it's just network requests.

Not connected to Google Drive yet.
Auto-backup to a local folder

Pick a folder once — FinPlan keeps finplan_autobackup.json updated there directly, no repeated prompts. Desktop Chrome/Edge only — Android/mobile browsers don't support this yet. Use "Auto-download backup" instead on mobile.

Auto-download backup

Works everywhere, including mobile/Android — periodically saves a dated JSON file straight to your device's Downloads folder using the ordinary browser download mechanism. Each save is a new file (it can't overwrite on mobile), so it runs every few hours rather than after every change — clear out old ones occasionally.

On Android/Chrome, install it like a native app — it opens in its own window with an icon on your home screen, and keeps working offline for data you've already loaded.

If you don't see an install button, use your browser menu → "Add to Home screen" / "Install app" (this is normal on iOS Safari, and on Android when the app is opened directly from a file rather than a hosted link).
